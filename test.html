Using CSS flexible boxes

The grid is based on the following HTML structure:

... using the following CSS block/element/modifiers:

  • grid: A grid element, i.e. a flex container. The grid element is only intended to be used as a wrapper for grid-cell elements (see below) and should not contain any actual content. In nested grids grid elements behave as grid-cell elements.
  • grid-cell: A grid cell element, i.e. a flex item. The cells are the elements that contain actual content.
  • grid--: A modifier class exclusively for grid elements.
  • g--: A modifier class for both grid and grid-cell elements.

In many of the following examples the grid--cell-count modifier class has been added to the wrapper grid elements. The modifier class causes underlying grid-cell elements to display their individual position among the total number of grid-cell elements in the given grid.

Fit, Shrink & Wrap Grid Cells

By default grid-cell (and child grid) elements stretch and divide the available space within a grid element equally, but you can change this by adding the modifier classes g--fit; g--shrink; and g--wrap to the individual grid-cell (and grid) elements. The modifier classes applies the following behavior of the element to which they are applied:

  • g--fit: Makes the element collapse (occupy the minimum amount of space) along the main axis, but retain its size on the cross axis.
  • g--shrink: Makes the element collapse on the cross axis, but retain its size on the main axis.
  • g--wrap: Makes the element collapse on both the main and cross axis, i.e. shrink-wrap the element to fit its contents.

Ordering

With flexbox you can change the order of appearance for flex-items without having to change the order of the HTML.

By default, each flex-item has an order: 0 (the equivalent of box-ordinal-group: 1 in older browsers/versions of Flexbox) - and the higher the order value, the farther back the element will be rendered. This means that if you only specify an order value higher than 0 on a single grid-cell in a collection (leaving the rest unspecified), the element with the order value higher than 0 will be rendered at the end of the collection, as the rest of the elements will have order: 0 by default.

CSS Pseudo Classes Not Respected

When using flexbox's reordering capabilities it seems that style-definitions for the CSS pseudo classes :first-child and :last-child are not respected as content is rendered according to the HTML element order.

The grid--gutters modifier class has been added to the grid elements to showcase the problem.
