Fit and Wrap

By default grid-cell elements stretch and divide the available space within a grid element equally, but you can change this by adding modifier classes to the individual grid-cell elements.

Fit

When you want to limit the width of a grid-cell element to fit its content, but still want it to expand to the height of the grid, you can apply the g--fit modifier class to the element itself.

Ordering (use with caution - see callout)

With flexbox you can change the order of appearance for flex-items without having to change the order of the HTML.

By default, each flex-item has an order: 0 (box-ordinal-group: 1 in older browsers) - and the higher the order value, the farther back the element will be rendered. This means that if you only specify an order value higher than 0 on a single grid-cell in a collection (leaving the rest unspecified), the element with the order value higher than 0 will be rendered at the end of the collection, as the rest of the elements will have order: 0 by default.

CSS Pseudo Classes Not Respected

When using flexbox's reordering capabilities it seems that style-definitions for the CSS pseudo classes :first-child and :last-child are not respected as content is rendered according to the HTML element order.
